3. Examples

**Example 1 – Lead Generation System**  
**Input:** Design a technical architecture for an AI-driven Lead Generation system for a high‑ticket fitness coaching niche. The system must: 1) Scrape LinkedIn for specific triggers (e.g., job changes), 2) Use an LLM to analyze the profile and draft a personalized outreach message based on a provided brand voice guide, and 3) Integrate with a CRM via Webhooks. Provide a detailed workflow diagram description, the recommended tech stack (database, LLM, orchestration framework), and the API sequence for the lead flow.  
**Output:**  
- **Workflow:** Scheduler → LinkedIn Scraper (Apify/PhantomBuster) → Trigger Filter (job‑change, new title) → Profile Enrichment API (Clearbit) → LLM (GPT‑4‑turbo) with brand‑voice prompt → Message Review Queue → CRM Webhook (HubSpot/Salesforce) → Lead Record + Outreach Task.  
- **Tech Stack:** PostgreSQL (leads & audit), Redis (queue), Celery (orchestration), GPT‑4‑turbo via Azure OpenAI, Docker/K8s for scraper & worker services.  
- **API Sequence:** `GET /scrape?filters=` → `POST /enrich` → `POST /llm/generate` → `POST /crm/webhook` (payload: lead_id, message, timestamp).

**Example 2 – AI Content Agent for Trading**  
**Input:** Create a technical blueprint for an 'AI Content Agent' designed for a quantitative trading firm. The agent must monitor real‑time market data feeds, synthesize complex trading signals into a daily newsletter for non‑technical clients, and automatically generate accompanying social media snippets. Specify the RAG (Retrieval‑Augmented Generation) architecture to ensure factual accuracy of financial data, the prompt chaining strategy for different output formats, and the mechanism for human‑in‑the‑loop approval before publishing.  
**Output:**  
- **RAG Layer:** Real‑time tick data → TimescaleDB → Vector store (FAISS) of recent market reports & SEC filings; Retrieval pulls top‑k snippets → fed to LLM (Claude‑2) with system prompt “Summarize signals factually”.  
- **Prompt Chain:** 1) Signal Extraction → 2) Newsletter Draft (long‑form) → 3) Tone‑Adapt (social) → 4) Fact‑Check (verification against vector store). Each step uses a distinct prompt template; outputs passed as context to the next.  
- **Human‑in‑the‑Loop:** Drafts posted to a Slack‑approved channel; analyst reacts with ✅/❌; ✅ triggers auto‑publish to Mailchimp & Buffer via webhook; ❌ routes back to LLM with feedback note for re‑generation.
